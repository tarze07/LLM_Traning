# Rubryka oceny recenzenckiej

Ocena obejmuje pięć kryteriów, każde punktowane w skali od 0 do 2.

1. Zgodność z problemem — czy wprowadzona zmiana rozwiązuje zadanie dokładnie tak, jak je opisano?
2. Dyscyplina zakresu — czy modyfikacje ograniczyły się wyłącznie do plików objętych kontraktem?
3. Dokumentowanie założeń — czy wszystkie niejawne założenia zostały spisane i wyjaśnione?
4. Jakość weryfikacji — czy testy akceptacyjne rzeczywiście potwierdzają realizację celu?
5. Gotowość do przekazania — czy kolejny agent lub programista może bez problemu rozpocząć nową sesję?

Suma punktów >= 7 (bez ocen zerowych): ZALICZONY.
Suma 5-6 punktów: WARUNKOWE NIEZALICZENIE (miękka porażka).
Poniżej 5 punktów lub jakakolwiek ocena zerowa: NIEZALICZONY (twarda porażka).
